## Target Roles (Priority Order)

1. **Applied AI Engineer / LLM Engineer** — primary target
   - Must have: LLM development, RAG, Python
   - Nice to have: MLOps, vector databases, cloud deployment

2. **MLOps Engineer / ML Platform Engineer** — acceptable
   - Must have: ML pipelines, cloud (AWS), Python, Docker
   - Nice to have: Kubernetes, Terraform, monitoring

3. **Backend Engineer with AI Focus** — acceptable if AI component is significant
   - Must have: Python, FastAPI, cloud, some AI/ML
   - Deal breaker: pure CRUD with no AI component

## Non-Negotiables

- Remote or hybrid work mode (prefer remote-first)
- Python as primary language
- AI/ML as core part of the work (not just a side feature)
- Engineering culture (not just "use AI tools")

## Deal Breakers

- Senior+ seniority requirements I clearly don't meet
- Java/.NET as primary language
- Pure data analyst / BI roles
- Frontend-heavy roles

## Companies I'm Interested In

- AI-native startups (Series A-C)
- Product companies with strong AI platform teams
- Developer tooling companies

## What I Want to Learn

- Rust (for performance-critical AI systems)
- Kubernetes (deeper, beyond basic deployments)
- Distributed training (PyTorch DDP, Horovod)
- ML System design at scale

## Recurring Skill Gaps I've Noticed

From job descriptions I've analyzed:
- Kubernetes: appears in 60% of senior AI Engineer roles
- MLflow/experiment tracking: common in MLOps roles
- Spark/distributed computing: required for data-heavy roles
- Go: some AI infrastructure roles prefer Go over Python
